# Research: Claude Code Agent/Skill Design, Multi-Agent Workflows, Framework Comparison
Date: 2026-04-20
Post: agent-design-physics-paper

## Key Facts

### Claude Code Architecture (Official)
- **Agents**: markdown files in `.claude/agents/` defining specialized roles with tool access; spawn separate conversations
- **Skills**: `SKILL.md` files with YAML frontmatter; preloaded into agent context, not spawned as separate conversations
- At startup, only skill name + description loaded into system prompt; full content loaded on-demand when relevant
- Frontmatter fields: `name`, `description`, `disable-model-invocation`, `user-invocable`, `allowed-tools`
- Source: [Claude Code Skills Docs](https://code.claude.com/docs/en/skills), [Skill Authoring Best Practices](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices)

### Agent Teams (Experimental, 2026)
- Requires Claude Code v2.1.32+, experimental flag `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS`
- One session acts as team lead; teammates work independently with own context windows
- Introduces peer-to-peer communication via mailbox system + shared task list (vs. subagents which are parent-to-child one-way)
- All agents in a team currently run the same model (Opus 4.6 required)
- Source: [Agent Teams Docs](https://code.claude.com/docs/en/agent-teams)

### Subagent Limitations (Confirmed)
- Subagents cannot spawn other subagents (parent-to-child one-way only)
- Context isolation: agents don't share full context window, only explicit messages
- If operator's context fills with too many subagent results, quality degrades
- Source: [Agent Teams Docs](https://code.claude.com/docs/en/agent-teams), [SmartScope comparison](https://smartscope.blog/en/blog/codex-vs-claude-code-subagent-architecture-2026/)

### Skill vs Agent Decision Framework
- **Use skills when**: task repeats across sessions, consistency matters, team standardization needed, behavior should be auditable
- **Use agents when**: task is one-time/unique, work can be parallelized, task exceeds single-agent scope, exploration with uncertain outcome
- Skills = "domain containers"; agents = "parallel workers that execute skills"
- Source: [Daniel Miessler](https://danielmiessler.com/blog/when-to-use-skills-vs-commands-vs-agents), [Sandeep Satya on Medium](https://medium.com/@SandeepTnvs/claude-skills-vs-sub-agents-architecture-use-cases-and-effective-patterns-3e535c9e0122)

### Skill Design Best Practices (Official)
- Two categories: "Capability Uplift" (new abilities) vs "Encoded Preference" (execute your way)
- Best development process: use one Claude instance ("Claude A") to create skills tested by another ("Claude B")
- Test skills across all models you plan to use
- Use gerund form for skill names (verb + -ing)
- Source: [Skill Authoring Best Practices](https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices)

## Competing Frameworks Comparison

### LangGraph (LangChain)
- Stateful graph orchestration with DAGs, conditional branching, parallel execution, durable state management
- Built-in checkpointing with time travel for state persistence
- Reached v1.0 late 2025; default runtime for all LangChain agents
- Best for: stateful workflows requiring branching and persistence
- Source: [DataCamp comparison](https://www.datacamp.com/tutorial/crewai-vs-langgraph-vs-autogen)

### CrewAI
- Role-based DSL; lowest learning curve (~35 lines minimal agent)
- Models collaboration as a "crew" of role-playing agents with roles, backstories, goals
- ~18% token overhead in 2026 benchmarks [VERIFY]
- Best for: team-based task automation, accessible entry point
- Source: [DataCamp comparison](https://www.datacamp.com/tutorial/crewai-vs-langgraph-vs-autogen)

### AutoGen/AG2 (Microsoft)
- v0.4 rewrite: event-driven core, async-first, GroupChat coordination
- October 2025: merged with Semantic Kernel into unified Microsoft Agent Framework
- AutoGen itself now in maintenance mode
- Best for: multi-agent conversations, research prototyping
- Source: [DataCamp comparison](https://www.datacamp.com/tutorial/crewai-vs-langgraph-vs-autogen)

### Claude Code (our approach)
- No framework: agents are markdown files, skills are markdown files, coordination is through notes directory
- Advantage: zero dependency, artifacts are inspectable text files, portable across projects
- Disadvantage: no built-in state management, no DAG orchestration, no checkpointing
- The "framework" is the file system + naming conventions + READMEs

## Patterns Relevant to the Blog Post

### Our approach vs frameworks
- We built a "frameworkless" multi-agent system: markdown definitions, file-system coordination, Obsidian for navigation
- This is closer to the Claude Code native approach than to LangGraph/CrewAI/AutoGen
- The key differentiator: skills as shared protocols solve the subagent delegation problem without requiring a framework's orchestration layer

### Novel patterns worth highlighting
- **Domain-scoped sharing via intermediate directory** (`~/Write/.claude/`): not in official docs, exploits directory-walking behavior
- **Agent auditor as self-improving system**: meta-agent that evaluates and improves the agent setup itself
- **Skills as team standardization**: all agents verify equations the same way, producing uniform artifacts
- **Notes directory as persistent memory**: replaces framework state management with plain files
