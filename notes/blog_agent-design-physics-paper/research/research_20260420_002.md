## Specific Researcher Workflows

### Patrick Mineault (Neuroscience, neuroai.science)
- "Claude Code for Scientists" guide for scientific coding workflows
- Key advice: separate data processing from visualization, use git branches for rollback, use plan mode before execution
- Argues Claude Code makes it feasible to be a solo researcher and remain productive as a senior PI
- Caution: code is generated faster than it can be verified; fundamentals (validation, reproducibility) don't change
- Source: [Claude Code for Scientists](https://www.neuroai.science/p/claude-code-for-scientists)

### Pedro Sant'Anna (Economics, Emory University)
- Ready-to-fork template: 28 skills, 14 specialized agents, 24 rules
- Adopted by 15+ research groups across economics, energy, political science, engineering
- Built for Econ 730 at Emory: produced 6 complete PhD lecture decks (800+ slides), interactive Quarto versions, full R replication packages
- Includes simulated peer review: editor desk review + 2 blind referees with different dispositions, calibrated to 5 econ journals (AER/QJE/JPE/ECMA/ReStud)
- Adversarial QA and quality gates built in
- Adapted into "Clo-Author" project for empirical economics research
- Source: [psantanna.com](https://psantanna.com/claude-code-my-workflow/), [GitHub](https://github.com/pedrohcgs/claude-code-my-workflow)

### Claude Scholar (CS/ML researchers)
- Semi-automated research assistant for Claude Code, OpenCode, Codex CLI
- Covers full lifecycle: literature review → ideation → coding/experiments → writing → publication
- Philosophy: human in the loop at every critical decision point
- Skills: literature-reviewer, research-ideation, ml-paper-writing, results-analysis, review-response
- Oriented toward ML papers and NeurIPS/ICLR/ICML conference circuit
- Source: [GitHub](https://github.com/Galaxy-Dawn/claude-scholar), [Menon Lab blog](https://themenonlab.blog/blog/claude-scholar-research-copilot-academic-writing)

## Patterns Across Disciplines

### What researchers consistently do
- Decompose research into specialized agents (editor, reviewer, writer, data analyst)
- Use persistent file-based memory (notes directories, cumulative databases)
- Keep human in the loop for critical decisions (not fully autonomous)
- Simulate peer review as a quality gate

### What varies by discipline
- Physics/math: emphasis on symbolic verification (SymPy, equation checking)
- Economics: emphasis on replication packages, R code, causal inference methods
- CS/ML: emphasis on experiment management, NeurIPS formatting, ablation studies
- Biology: emphasis on tool chaining (bioinformatics pipelines), data integration

### Common failure modes
- Code generated faster than it can be verified (Mineault)
- Hallucinated results and citation hallucinations (AI Scientist)
- Over-reliance on AI judgment for domain-specific decisions
- Loss of context across long sessions (motivates persistent notes)
