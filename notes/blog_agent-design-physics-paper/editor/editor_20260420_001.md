## 3. Style Check

- **Line 82**: "These are exactly the inconsistencies that accumulate during a multi-day writing process and that a human re-read tends to miss because you remember what you intended to write rather than what you actually wrote." — 38-word sentence, could be tightened. The "because you remember what you intended" clause is the insight; the setup is filler.
- **Line 88**: "This is not about independent discovery." — Declarative summary sentence announcing what follows. The next sentence ("The typical trigger is an impasse...") does the work; this sentence can be cut.
- **Line 132**: "The solution was to convert shared protocols from agents into skills." — Passive "was to convert." Active: "We converted shared protocols from agents into skills."
- **Line 148**: "This turned out to be better than the original delegation design." — Throat-clearing. Lead with the reason.
- **Line 158**: "Every agent writes to notes/. The editor writes reading logs. The grad student writes investigation notebooks. The researcher writes search logs and a cumulative database. The verification skill produces scripts with a README. The figure skill produces scripts with a README." — Six short sentences listing who writes what. This was covered in the Architecture section's directory tree. Redundant.

## 4. Consistency Check

- Agent subsections have inconsistent structure. Editor has: how it works → usage pattern + results. Grad Student has: purpose → method → example 1 → example 2. Researcher has: method → results. Writer has: method + rationale. Agent Auditor has: purpose → one result. Not a problem per se (each agent is genuinely different), but the Editor and Grad Student get concrete results while the Writer gets none.
- "What Worked" entries use `<strong>Title.</strong>` followed by prose. "What Didn't" entries use the same format. Consistent.
